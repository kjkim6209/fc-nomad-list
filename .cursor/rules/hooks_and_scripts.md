## 커스텀 훅 (`hooks/`)

- **네이밍 & 책임**
  - 파일 이름은 `use-*.ts` 형태를 유지한다 (예: `use-mobile.ts`, `use-toast.ts`).
  - 커스텀 훅은 **단일 책임 원칙**을 지키고, UI에 의존하지 않는 순수 로직/상태 관리에 집중한다.
  - 훅은 반드시 `use` 접두사를 사용하고, React 훅 규칙(조건문 안에서 호출 금지 등)을 준수한다.

## 스크립트 (`scripts/`)

- **역할**
  - Node.js 실행을 전제로 하며, 현재 프로젝트에서는 주로 **빌드/유틸성 스크립트**(예: `download-city-images.mjs`)를 둔다.
  - 스크립트는 **비상호작용(non-interactive)** 모드에서도 실행 가능하도록 설계한다 (CLI 입력에 과도하게 의존하지 않는다).
  - 외부 API/네트워크를 사용하는 경우 에러 처리와 로그 출력을 명확히 하여 문제 상황을 추적하기 쉽게 한다.

## 공통 로직 & 재사용

- **공통**
  - 재사용 가능한 로직은 `lib/utils.ts`에 두고, hooks나 scripts에서 공유해 사용한다.
  - 새 훅/스크립트를 만들 때는, 동일 역할을 하는 기존 구현이 있는지 먼저 확인한 뒤 중복을 피한다.
